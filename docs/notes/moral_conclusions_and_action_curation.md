# Note: Unrealistic Moral Conclusions and the Role of Action Curation

*Article material. Connects to item 15 (framing problem) and the critical evaluation of AS1/AS2.*

---

## The observation

Several arguments generated by the system are formally valid but morally implausible. For
example, AS2 arguments can be constructed for action sequences that a mature moral reasoner
would immediately rule out — not because they fail to protect a value, but because the
action itself is intrinsically prohibited regardless of its consequences. The framework has
no mechanism to represent this kind of constraint.

The question is whether this is a bug in the implementation or a documented limitation of
the framework. It is the latter. But acknowledging it is important for the article.

---

## The handpicked transition problem

In Atkinson & Bench-Capon (2006) and the subsequent literature, the action set, precondition
function, and transition function are **selected by the modeller** prior to any formal
reasoning. The insulin scenario includes precisely the actions needed to make the framework
illustrative: `buyH`, `comH`, `takH`, `losH`, `doNH`. Actions that would generate absurd or
morally repugnant arguments simply do not appear — not because the framework excludes them,
but because the modeller never included them.

This is not disclosed as a methodological choice in the original paper. It is presented as
a natural formalisation of the domain. But the formalisation is doing significant normative
work upstream of the logic: by deciding which actions exist, the modeller determines which
arguments can be constructed. The formal system then endorses or rejects those arguments —
but it cannot question whether the action itself should be in the model.

This is a direct instance of the frame problem (see `docs/notes/framing_problem.md`), but
applied to transitions rather than propositions. The question "which state transitions count
as legal actions?" is answered prior to the formal system, not by it. The logic inherits
the modeller's prior normative commitments without making them explicit.

### Consequence for our implementation

When we extended the action set (joint actions, `earnH`, AS2 over all values), we
encountered arguments that are formally generated but morally thin. This is expected: the
extension of the domain expands the space of formally defensible positions without any
corresponding expansion of normative filtering. The framework does not distinguish between
"Hal compensates Carla" and "Hal takes Carla's insulin by force" as actions — both are
nodes in the transition system, both can appear in arguments, both can be endorsed by some
preferred extension. The moral difference is invisible to the logic.

---

## The consequentialist constraint of AS1 and AS2

The deeper issue is structural. Both argument schemes are purely **consequentialist**:

- **AS1**: In circumstances R, perform A, which leads to state S, realising goal G, promoting
  value V. The argument is valid if A promotes V. The intrinsic nature of A is irrelevant.
- **AS2**: In circumstances R, perform A, because not doing so would lead to S, which would
  demote V. Again, A is evaluated only by its protective effect on V.

There is no room in either scheme for:

- **Deontological constraints**: "Do not do X regardless of its consequences." No action is
  ruled out a priori on grounds of rights-violation, dignity, or duty.
- **Agent-relative reasons**: "The fact that I would be the one performing the harmful action
  makes a moral difference." The scheme is agent-neutral with respect to action type.
- **Threshold effects**: "Even if doing X would promote V, V cannot be promoted by X beyond
  a certain point without becoming impermissible." The scheme has no upper bound on
  value-promotion as a justification.

This means the framework systematically underpopulates the space of moral constraints. It
correctly captures one important class of moral reasoning — the deliberation of a benevolent
consequentialist agent — but it misrepresents the structure of moral reasoning more broadly
by omitting the side-constraints that most ethical theories recognise.

---

## Is this a criticism of the paper or of the implementation?

Both, proportionally.

**Of the paper**: Atkinson & Bench-Capon (2006) present the framework as a model of
"practical reasoning" without adequately flagging its consequentialist commitments or the
normative work done by the modeller in selecting the action set. The framework is positioned
as domain-general, but its outputs are only as morally serious as the domain specification
it is given. This should be stated explicitly.

**Of implementations** (including ours): Any implementation that extends the action set —
especially by adding AS2 or joint actions — will generate more formally valid arguments,
not more morally serious ones. The ratio of formally defensible to morally serious positions
may actually decrease as the domain grows, because formal validity is easier to achieve than
moral plausibility.

---

## What this means for the article

The article should make the following distinction explicit:

> The system maps the space of *formally defensible* positions under a given value ordering
> and action set. It does not map the space of *morally serious* positions. The gap between
> the two is not closed by the logic — it requires prior normative commitments about which
> actions are permissible, which the framework cannot represent and which the modeller must
> supply by hand.

This is not a fatal objection to the framework. It is a precise characterisation of its
scope. A decision support tool that makes consequentialist deliberation explicit and
transparent has genuine value. The objection is only fatal if the framework is presented
as a complete model of moral reasoning — which it should not be.

The critical contribution of the article is to make this distinction clearly, using our
implementation as evidence: we followed the formal specification faithfully, extended it
consistently, and the results reveal exactly where formal correctness and moral plausibility
come apart.
